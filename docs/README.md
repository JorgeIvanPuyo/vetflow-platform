# Documentación del proyecto

Esta carpeta reúne documentación de producto, arquitectura y planificación para el equipo. Puede incluir propuestas e información histórica; el código, las pruebas, las migraciones y los manifests determinan el comportamiento técnico vigente.

Los documentos nuevos o modificados se escriben en español. El material técnico histórico se traduce sólo cuando una tarea lo actualice.

| Si la tarea trata sobre | Consulta |
| --- | --- |
| Alcance, prioridades o una nueva funcionalidad | [Alcance](product-scope.md), [roadmap](roadmap.md), PRD o backlog pertinente |
| Arquitectura o decisiones transversales | [Arquitectura](architecture-overview.md) y decisión técnica pertinente |
| Backend, modelos, migraciones o pruebas | [Convenciones backend](backend-conventions.md) |
| Rutas, componentes, UX o tipos web | [Convenciones frontend](frontend-conventions.md) |
| Endpoints, envolturas de respuesta o compatibilidad | [Contratos API](api-contracts.md) |
| Aislamiento de clínicas, roles o datos compartidos | [Multitenancy](multitenancy-strategy.md) |
| Pipelines, despliegues o secretos | [CI/CD](cicd-strategy.md) |
| Almacenamiento de archivos clínicos | [Archivos clínicos](clinical-file-storage-architecture.md) |

## Estado y planificación

- [roadmap.md](roadmap.md) es la única fuente activa del estado del MVP y de las prioridades técnicas.
- [prd/](prd/) contiene especificaciones de producto por iniciativa.
- [backlog/](backlog/) contiene investigación, priorización y material de validación pendiente.
- [archive/](archive/) conserva planes históricos; no los uses para decidir el estado actual.
